Developers¶

The Origen framework exists in two parts: a Python-based frontend, which users will build their applications upon, and a compiled backend, which maintains the device, tester, and other various models.

Development Installation¶

Follow as shown the installation guide here to setup your development environment.

User API¶

The origen module is the user-facing frontend.

Development APIs¶

Not disclosed to the users is the _origen API which functions as an adapter connecting the frontend user API origen module to the Rust-powered backend.

This API should not be used by users directly and is entirely derived from the Rust backend. Rather, functions exposed to the users through the origen module should encapsulate all calls to _origen. This includes simple getters, such as origen.app.output_dir, whose implementation is merely a call to the _origen.output_directory() function.

The backend is split into two Rust libraries: PyAPI and Origen. PyAPI is a PyO3 library which implements _origen.

APIs¶

  • The user API: origen module

  • The frontend adapter to the backend, _origen

  • The backend listener, pyapi

  • The compiled backend model, also called origen
