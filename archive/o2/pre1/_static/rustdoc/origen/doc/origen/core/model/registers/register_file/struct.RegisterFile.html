pub struct RegisterFile {
    pub id: usize,
    pub address_block_id: usize,
    pub register_file_id: Option<usize>,
    pub name: String,
    pub description: String,
    pub dim: u32,
    pub offset: u128,
    pub range: u64,
    pub registers: IndexMap<String, usize>,
    pub register_files: IndexMap<String, usize>,
}

Represents a groups of registers within an address block. RegisterFiles can also contain other RegisterFiles.

Fields

id: usizeaddress_block_id: usizeregister_file_id: Option<usize>

Optional, if this register file is a child of another register file then its parent ID will be recorded here

name: Stringdescription: Stringdim: u32

The dimension of the register, defaults to 1.

offset: u128

The address offset from the containing address block or register file, expressed in address_unit_bits from the parent memory map.

range: u64

The number of addressable units in the register file.

registers: IndexMap<String, usize>register_files: IndexMap<String, usize>

pub fn friendly_path(&self, dut: &MutexGuard<Dut>) -> Result<String>[src]

Returns a path to this register file like "dut.my_block.my_map.my_address_block.my_reg_file", but the map and address block portions will be inhibited when they are 'default'. This is to keep map and address block concerns out of the view of users who don't use them and simply define regs at the top-level of the block.
