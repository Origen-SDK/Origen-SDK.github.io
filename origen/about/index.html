About RGen

For test engineers RGen is a full-stack test engineering framework, meaning that it can be used it to generate test patterns, test programs, simulation patterns and documentation. For everyone else RGen is a general purpose framework which will help you to easily generate and manage derivative-specific documentation and IP of any kind.

RGen was initially conceived and developed by the C90TFS NVM test engineering team who created it as the foundation on which to build the next generation of test environments to support Freescale’s NVM technologies. It is now the standard technology stack for all of the NVM test environments developed at Freescale and it is primarily maintained and developed by the NVM test engineering team.

However RGen is specifically designed to be technology-agnostic and there is nothing NVM-specific about it. In fact as it turns out the needs of the NVM team are not too dissimilar to those from any other discipline and RGen is starting to be picked up by a growing number of non-NVM applications.

RGen has also been used successfully within the NVM design team to pre-compile RTL and firmware where RGen’s object-oriented configuration management really helped to bring order to a derivative and bug tracking landscape which was growing out of control.

You can read some more about that here: Taming Compiler Directives with RGen

To learn more please consult the RGen Guides, we think you will like it.
